Prototype variables

This prototype uses variables to enable the scenarios to work. If in doubt, re-set the prototype using the 'Clear data' button at the bottom this page.

For example a url for first time registration. Sets the service to be Comply with REACH, the userType to be a new user, no extra service questions and registration ends with the user approved
https://identity-management-app.herokuapp.com/service-start-logged-out?serviceLine=Comply with UK REACH&serviceRole=REACH Manager&userType=new®Success=success&serviceQuestions=no

Registration/ Enrolment/ Add account

userType= new / existing / invited / invitedAgent
What is the type of user?

regSuccess= pending / success / incomplete
What is the enrolment status at the end of registration (success = 'approved' in dynamics)

serviceQuestions= yes / no
Are there any service questions once the IDM questions are complete?

businessType= Limited company / Sole trader / Public limited company / Limited liability partnership / Charity
What is the business type being registered?

flowType= task / check
What stage of the reg flow are they in: tasklist or check answers?

systemRole= Admin / Standard user
What is the users IDM account role?

serviceRole= REACH Manager / REACH Reader / Manager / Standard etc
Or:
serviceRole= No service access / Pending / Rejected / Incomplete

What is the status of the users enrolment to a service? If the user is 'approved' then show the specific servie role, else show the service status

General

userType= new / existing / invited / invitedAgent
What is the type of user?

numUsers= 1 - 6
The number of team members in an account

numServices= 1 - 3
The number of services on the 'Your Defra account' or the account dashboard

numAgents= 0 - 3
The number of intermediaries in an account)

numLocations= 1 - 3
How many orgs on a multi-org dashboard

serviceLine= Comply with UK REACH
What is the specific service name, can use any example

orgName= Acme Org Ltd
What is the parent org called

childName= Chemicals R US
What is the child org called

orgType= Business / Individual / onlyRep
What is the type of organisation? - use for determining the Your Defra account page and top navigation

Invite / n+1

accountAccess= yes / no

routes.js

The routes.js files determines different versions of a template according to a specific journey / ser type. Rather than have 4 versions of a page, there is one template that has different variables according to the journey. This has been applied to the following templates:

  • SCP login page
  • Account dashboard
